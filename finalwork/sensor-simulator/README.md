# sensor-simulator：模拟温湿度采集节点

## 功能特性

- **数据生成**：生成温度、湿度、时间戳、设备编号、序号
- **噪声模拟**：加入随机白噪声，模拟真实传感器误差
- **异常模拟**：
  - 跳变异常：随机触发大幅数据跳变
  - 丢包模拟：随机跳过数据发送
  - 离线模拟：模拟设备短暂离线
- **MQTT 发布**：通过 Paho-MQTT 发布 JSON 数据
- **断线重连**：指数退避自动重连机制
- **可视化订阅端**：Python Tkinter + Matplotlib 实时曲线界面

```bash
pip install -r requirements.txt
```

## 运行模式

### 1. 发布端 (Publisher) - 模拟传感器

**普通模式（随机异常）**：
```bash
python mqtt_publisher.py --mode normal
```

**演示模式（固定序列，方便展示）**：
```bash
python mqtt_publisher.py --mode demo
```

### 2. 订阅端 (Subscriber) - 可视化界面

```bash
python mqtt_subscriber_gui.py
```

启动后会自动连接 MQTT Broker 并显示：
- 实时温湿度数值（大字体显示）
- 动态曲线图（每秒更新）
- 连接状态指示
- 报警信息面板

## 数据格式

发送的 JSON 数据：

```json
{
  "deviceId": "env-node-001",
  "seq": 1001,
  "timestamp": "2026-06-16 20:35:01",
  "temperature": 25.4,
  "humidity": 60.1,
  "status": "normal"
}
```

### 数据状态说明

| 状态 | 说明 | 视觉效果 |
|------|------|----------|
| `normal` | 正常数据 | 绿色 |
| `jump` | 跳变异常 | 黄色警告 |
| `temp_high` | 高温报警 | 红色报警 |
| `humidity_high` | 高湿报警 | 红色报警 |
| `both_high` | 复合报警 | 红色报警 |
| `lost` | 丢包 | 本次不发送 |
| `offline` | 设备离线 | 离线提示 |

## 演示脚本（推荐录屏顺序）

### 视频录屏脚本（2分钟）

| 时间 | 操作 | 预期效果 |
|------|------|----------|
| 0-10s | 启动 Publisher (demo模式) | 控制台显示彩色日志 |
| 10-20s | 启动 Subscriber GUI | 曲线开始绘制 |
| 20-30s | 观察正常数据 | 绿色状态，曲线平稳 |
| 30-40s | 等待触发跳变 | 黄色报警，曲线突变 |
| 40-50s | 等待触发高温 | 红色报警，卡片变红 |
| 50-60s | 观察重连机制 | 关闭 Broker 后显示重连状态 |

## 配置文件

修改 `config.py`：

```python
MQTT_HOST = "broker.emqx.io"      # MQTT Broker 地址
MQTT_PORT = 1883                  # MQTT 端口
MQTT_TOPIC = "iot/env/node001/data"  # 发布主题

# 数据生成配置
BASE_TEMPERATURE = 26.0           # 基础温度
BASE_HUMIDITY = 60.0              # 基础湿度
NOISE_STD = 0.5                   # 噪声标准差

# 异常概率
JUMP_PROBABILITY = 0.08    # 跳变异常概率 8%
DROP_PROBABILITY = 0.05      # 丢包概率 5%
OFFLINE_PROBABILITY = 0.02  # 离线概率 2%
```

## 断线重连机制

当连接断开时，采用指数退避策略重连：

1. 初始延迟：1 秒
2. 最大延迟：30 秒
3. 退避倍数：2 倍
4. 持续重试直到成功

控制台会显示：
```
[INFO] 连接异常: [Errno 111] Connection refused，1s 后重试 (第 1 次尝试)
[INFO] 连接异常: [Errno 111] Connection refused，2s 后重试 (第 2 次尝试)
...
[OK] 重连成功！Broker: broker.emqx.io:1883 (第 5 次重连)
```
