# 运行说明

## 一、Docker 一键部署方式

本项目已经提供 `docker-compose.yml`，可一键启动 MQTT Broker、Spring Boot 后端、Vue 前端和 Python 模拟传感器。

### 1. 安装 Docker Desktop

Windows / macOS 均可安装 Docker Desktop。安装完成后，在项目根目录执行：

```bash
docker compose up -d --build
```

### 2. 访问系统

- 前端展示页：http://localhost:5173
- 后端健康检查：http://localhost:8080/api/health
- EMQX 管理后台：http://localhost:18083
- EMQX 默认账号：admin
- EMQX 默认密码：public

### 3. 停止系统

```bash
docker compose down
```

## 二、本地分别启动方式

### 1. 启动 Spring Boot 后端

```bash
cd iot-backend
mvn spring-boot:run
```

### 2. 启动 Vue 前端

```bash
cd iot-frontend
npm install
npm run dev
```

浏览器访问：

```text
http://localhost:5173
```

### 3. 启动 Python 模拟传感器

```bash
cd sensor-simulator
pip install -r requirements.txt
python mqtt_publisher.py --mode demo
```

`demo` 模式会按固定顺序触发正常、跳变、超温、超湿、丢包、离线等场景，方便课堂演示。

## 三、常见问题

### 1. 前端没有数据

检查后端是否启动成功，访问：

```text
http://localhost:8080/api/health
```

检查模拟器是否在持续输出 `[PUB]` 日志。

### 2. MQTT 无法连接

如果使用公共 Broker 不稳定，可改用 Docker 内置 EMQX，或者修改配置：

- Python：`sensor-simulator/config.py`
- 后端：`iot-backend/src/main/resources/application.yml`

### 3. WebSocket 未连接

确认后端 8080 端口已启动，并且前端代码中的 WebSocket 地址为：

```text
ws://localhost:8080/ws/sensor
```
