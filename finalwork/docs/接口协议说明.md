# 接口协议说明

## 一、MQTT Topic

```text
iot/env/node001/data
```

## 二、传感器上报 JSON

```json
{
  "deviceId": "env-node-001",
  "seq": 1001,
  "timestamp": "2026-06-15 20:30:00",
  "temperature": 28.6,
  "humidity": 65.2,
  "status": "normal"
}
```

## 三、后端推送给前端的数据

```json
{
  "deviceId": "env-node-001",
  "seq": 1001,
  "timestamp": "2026-06-15 20:30:00",
  "rawTemperature": 38.5,
  "filteredTemperature": 36.8,
  "rawHumidity": 82.4,
  "filteredHumidity": 79.5,
  "rawStatus": "temp_high",
  "deviceStatus": "online",
  "outlier": false,
  "packetLoss": false,
  "alarmType": "TEMP_HIGH",
  "alarmMessage": "温度超过阈值",
  "diagnosisAdvice": "建议检查设备是否靠近热源、机房散热是否正常，并确认温度传感器未被遮挡。"
}
```

## 四、REST 接口

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/health` | GET | 后端健康检查 |
| `/api/latest` | GET | 获取最新处理后数据 |
| `/api/history` | GET | 获取历史数据 |
| `/api/alarms` | GET | 获取报警记录 |

## 五、WebSocket

```text
ws://localhost:8080/ws/sensor
```

后端每收到一条 MQTT 数据并完成处理后，会通过 WebSocket 推送给前端。
