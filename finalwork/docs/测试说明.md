# 测试说明

## 一、正常数据测试

启动模拟器 `demo` 模式，前端应能持续显示温度、湿度变化曲线。

预期结果：

- WebSocket 显示已连接；
- 当前温度、当前湿度持续更新；
- 温度和湿度曲线动态变化。

## 二、超温报警测试

`demo` 模式会自动生成 `temp_high` 状态。

预期结果：

- 报警状态显示 `TEMP_HIGH`；
- 报警记录列表新增记录；
- 前端显示诊断建议。

## 三、超湿报警测试

`demo` 模式会自动生成 `humidity_high` 状态。

预期结果：

- 报警状态显示 `HUMIDITY_HIGH`；
- 报警记录列表新增记录；
- 前端显示湿度异常诊断建议。

## 四、跳变异常测试

`demo` 模式会自动生成 `jump` 状态。

预期结果：

- 后端将数据标记为 outlier；
- 异常点不进入滑动平均窗口；
- 报警类型显示 `DATA_OUTLIER`；
- 前端显示传感器接触不良或采样干扰相关建议。

## 五、丢包测试

`demo` 模式会自动跳过一次数据发布。

预期结果：

- 后端通过 seq 序号不连续识别丢包；
- 前端最近数据中 `丢包` 字段显示“是”；
- 报警记录显示 `PACKET_LOSS`。

## 六、断线重连测试

可临时停止 EMQX 或断开网络，观察日志。

预期结果：

- Python 发布端进入指数退避重连；
- Spring Boot 后端使用 Paho 自动重连；
- Broker 恢复后，系统继续接收和展示数据。
