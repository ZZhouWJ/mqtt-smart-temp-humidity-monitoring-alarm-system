# 工程化改进说明

本次改进主要围绕“稳定演示”和“工程规范”展开，而不是盲目堆技术。

## 1. Docker 一键部署

项目新增 `docker-compose.yml`，可以同时启动：

- EMQX：本地 MQTT Broker；
- iot-backend：Spring Boot 后端；
- iot-frontend：Vue 前端；
- sensor-simulator：Python 模拟传感器。

这样可以避免课堂演示时公共 MQTT Broker 不稳定或环境依赖复杂的问题。

## 2. 本地规则诊断模块

Gemini 建议接入 LLM API 生成智能诊断。考虑到 API Key、网络和费用问题，本项目采用本地规则诊断模块 `DiagnosisService`。

它可以根据报警类型生成诊断建议：

| 报警类型 | 诊断建议示例 |
|---|---|
| TEMP_HIGH | 检查热源、通风和传感器遮挡 |
| HUMIDITY_HIGH | 检查漏水、潮湿和通风不足 |
| DATA_OUTLIER | 检查传感器接触不良或电磁干扰 |
| PACKET_LOSS | 检查网络质量和 MQTT 连接 |
| DEVICE_OFFLINE | 检查采集程序和设备连接 |

该模块不依赖外部 API，演示稳定；后续可替换为 DeepSeek、Kimi 等大模型接口。

## 3. 时序数据库拓展设计

当前系统采用内存缓存最近数据和报警记录，适合课程演示。真实物联网场景下，数据具有高频写入和时间窗口查询特征，后续可接入 InfluxDB 或 TDengine。

预留拓展方向：

```text
SensorDataService → TimeSeriesRepository → InfluxDB / TDengine
```

这样可以支持：

- 长期历史数据存储；
- 按分钟、小时、天聚合查询；
- 温湿度趋势分析；
- 大规模设备接入后的高吞吐写入。
