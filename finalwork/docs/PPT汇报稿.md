# PPT 汇报稿

## 开场

各位老师好，我们小组选择的是题目一：基于 MQTT 协议的模拟环境监测物联网系统。我们将它工程化实现为一个智能温湿度环境监测与异常报警系统，完整覆盖模拟采集、MQTT 上传、后端处理、前端展示、异常处理和断线重连。

## 系统架构

系统分为四层：感知层使用 Python 模拟温湿度传感器；网络层使用 MQTT Broker 进行发布订阅；平台层使用 Spring Boot 订阅数据并完成滤波、丢包检测和报警判断；应用层使用 Vue 和 ECharts 实时展示曲线和报警记录。

## MQTT 选择理由

我们没有直接使用 TCP Socket，而是选择 MQTT。TCP 是点对点连接，需要自己设计应用层协议和消息分发机制。MQTT 基于发布订阅模式，通过 Topic 解耦设备端和后端，更适合物联网中大量设备上报小体量数据的场景。本项目采用 QoS 1，在可靠性和通信开销之间取得平衡。

## 数据格式

我们采用 JSON 数据格式，字段包括 deviceId、seq、timestamp、temperature、humidity 和 status。其中 seq 用于丢包检测，status 用于标记正常、跳变、丢包和离线等状态。JSON 可读性强，Python、Java 和 Vue 都可以方便解析，也便于后续扩展更多传感器字段。

## 异常处理

项目主动模拟随机噪声、跳变异常、丢包异常和离线异常。后端使用滑动平均滤波降低随机噪声影响，通过阈值判断识别跳变异常，通过 seq 序号识别丢包。异常点不会进入滑动窗口，避免污染后续曲线。

## 工程化改进

在基础功能之外，我们新增了 Docker 一键部署和本地规则诊断模块。Docker 让老师可以一键启动 EMQX、后端、前端和模拟器；诊断模块根据报警类型生成维护建议，不依赖外部 LLM API，保证演示稳定。后续可扩展时序数据库和大模型诊断。

## 结尾

最终系统实现了完整的物联网数据链路，并且能稳定演示实时曲线、超温超湿报警、异常数据处理、断线重连和诊断建议生成。以上就是我们的项目汇报。
