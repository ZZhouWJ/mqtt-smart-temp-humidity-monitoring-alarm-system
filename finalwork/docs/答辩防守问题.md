# 答辩防守问题准备

## 1. 滑动平均滤波有什么缺陷？

滑动平均滤波可以降低随机噪声，让曲线更平滑，但它存在滞后性。如果真实环境温度快速上升，滑动窗口会让变化反映得稍慢。

本项目的处理方式是：同时保留原始值和滤波值，并在后端设置跳变阈值。异常点会被标记为 outlier，不进入滑动窗口，避免污染后续曲线。前端也会同时显示原始曲线和滤波曲线，便于观察数据处理效果。

## 2. MQTT 的 QoS 选择几？为什么？

本项目发布端和订阅端均使用 QoS 1。

QoS 1 表示“至少送达一次”。温湿度数据是高频时序数据，偶尔重复一条数据可以通过 seq 字段和后端处理容错；但如果使用 QoS 0，消息可能直接丢失。QoS 2 虽然可靠性更高，但握手成本更大，不适合高频采集数据。因此本项目选择 QoS 1，在可靠性和性能之间取得平衡。

## 3. 为什么不用 TCP？

TCP 是更底层的点对点传输协议，若直接使用 TCP，需要自己设计应用层协议、连接维护、多客户端分发和重连逻辑。MQTT 基于 TCP，但提供发布/订阅模型、Topic 解耦、QoS 机制和保活机制，更适合物联网设备数据上报。

## 4. 为什么 JSON 中要加 seq 字段？

seq 是数据序号。后端通过判断 seq 是否连续来发现丢包。例如上一条数据 seq=1001，下一条数据 seq=1004，则说明中间可能丢失了 1002 和 1003。这样可以把“丢包检测”从抽象描述落实到代码逻辑中。

## 5. 为什么没有真正接入 LLM API？

本项目以物联网通信、异常处理和实时展示为核心。为了保证课堂演示稳定，系统采用本地规则诊断模块，不依赖外部 API Key 和网络服务。后续可把 DiagnosisService 替换为大模型 API，实现更自然的异常解释。
